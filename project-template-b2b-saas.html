project-memory.md — Décisions architecturales

// Chaque agent AI lit ce fichier avant de produire du code. Toute nouvelle décision y est ajoutée.

DEC-001
Authentification via JWT RS256 — tokens signés asymétriquement, révocables via Redis.
// ISO A.9 · SOC2 CC6 · Choisi vs HS256 pour permettre la vérification sans secret partagé entre services
DEC-002
Cache utilisateur via Redis avec TTL 15 minutes. Invalidation sur modification de rôle.
// Perf: évite N+1 sur chaque requête auth. Sécurité: invalidation forcée si révocation
DEC-003
Communication asynchrone inter-services via Kafka. REST uniquement pour les appels synchrones critiques.
// Découplage + audit trail naturel via event store. Kafka choisi vs RabbitMQ pour la rétention des messages
DEC-004
Isolation données : PostgreSQL par service + Row-Level Security (RLS) sur tenant_id.
// SOC2 CC6 · Boundary forte entre tenants. RLS = dernier rempart même si le code applicatif échoue
DEC-005
Tous les secrets via HashiCorp Vault — rotation automatique 90j. Aucun secret dans le code ou CI/CD.
// ISO A.10 · SOC2 CC6 · Vault audit log intégré au SIEM
DEC-006
Service Mesh Istio avec mTLS obligatoire entre tous les pods. Network policies K8s deny-all par défaut.
// Zero Trust Network · ISO A.13 · mTLS = chiffrement + authentification mutuelle entre services
DEC-007
Logs d'audit WORM (Write Once Read Many) avec rétention 1 an minimum, exportés vers Datadog.
// SOC2 CC7 · Immuabilité garantie pour les audits externes. Format OpenTelemetry standardisé
DEC-008
GitOps strict via ArgoCD — aucun déploiement manuel en prod. Chaque PR déclenche SAST + scan image.
// ISO A.14 · Traçabilité totale des déploiements. Images signées Cosign avant push vers registry
Business Rules (business-rules.md)
BR-001
Un utilisateur appartient à un seul tenant. Cross-tenant interdit même pour les super-admins.
BR-002
Toute action d'un admin doit générer une entrée d'audit immuable (qui, quoi, quand, depuis où).
BR-003
Un compte désactivé doit voir ses tokens révoqués immédiatement (Redis invalidation + Kafka event).
BR-004
Les données sont la propriété du tenant. En cas de résiliation, export dans les 30j puis suppression certifiée.
